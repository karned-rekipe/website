# Comprendre Rekipe

## Le problème

Organiser les repas demande beaucoup de petites décisions : trouver une idée, vérifier les ingrédients, prévoir les quantités, anticiper une préparation, puis faire les courses.

Quand la base de recettes devient difficile à maintenir, tout le système se dégrade : on ajoute moins de recettes, on réutilise moins bien l'historique, et la planification devient plus pénible.

## La réponse produit

Rekipe structure le sujet en trois volets.

| Volet | Rôle | Exemple d'usage |
|---|---|---|
| Recettes | Conserver une mémoire culinaire fiable | "Je veux retrouver et modifier une recette rapidement." |
| Planification | Construire les repas à venir | "Qu'est-ce qu'on mange cette semaine ?" |
| Courses | Transformer les repas en achats | "Qu'est-ce qu'il faut acheter, et où ?" |

Ces trois volets sont liés mais séparés. Une recette n'est pas un repas. Un ingrédient n'est pas directement un produit à acheter. Une liste de courses n'est pas un stock.

## Le rôle de l'IA

L'IA est utile quand elle réduit une charge concrète :

- aider à créer une recette depuis une source ;
- suggérer des repas cohérents avec les saisons et l'historique ;
- contrôler une liste de courses et repérer les incohérences.

Elle ne remplace pas la validation humaine. La vérité métier reste dans les services applicatifs et les données validées.

## Les priorités produit

Quand plusieurs objectifs se contredisent, l'ordre de décision est :

1. anti-gaspillage ;
2. santé ;
3. saisonnalité ;
4. variété ;
5. praticité.

Ce choix donne une direction simple au produit : proposer une organisation qui aide vraiment le foyer, sans se transformer trop tôt en moteur d'optimisation complexe.

## Ce que l'utilisateur doit retenir

Rekipe doit devenir le cockpit simple des repas :

- une base de recettes propre ;
- une semaine de repas lisible ;
- une liste de courses exploitable ;
- des suggestions utiles, mais toujours contrôlables.
